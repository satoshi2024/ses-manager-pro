# 正式データバックアップ・復元 要件

## 目的
本番データベースと契約書を含む uploads を暗号化された異地保管へ退避し、障害時に指定時点まで安全に復元できるようにする。

## 要件

### 要件 1：バックアップ
- 毎日 02:00（Asia/Tokyo）に `mysqldump --single-transaction` による全バックアップを作成する。
- dump、uploads、manifest（Flywayバージョン・ファイル一覧・SHA-256）を restic で S3 互換リポジトリへ暗号化保存する。
- 保存期間は30日、バックアップ用認証情報は Docker secrets または環境変数ファイルから読み、ログ・引数に出さない。

### 要件 2：ポイントインタイム復元
- `mysqlbinlog --read-from-remote-server --raw --stop-never` で binlog を継続収集し、15分以内のRPOを目標とする。
- 全バックアップと連続した binlog を指定時刻まで検証し、最初に安全スナップショットを作成してから `--apply` 明示時のみ復元する。
- 復元後にスキーマ履歴、主要テーブル件数、uploads ハッシュを検証し、RTO目標は4時間以内とする。

### 要件 3：監視と演習
- 全バックアップが26時間超、binlog同期が30分超の場合に非0終了するヘルスチェックを提供する。
- 隔離MySQLコンテナへの四半期リストア演習スクリプトと手順を提供する。
- Web画面からの一発復元は提供せず、運用者が停止・確認・実行の二段階で行う。
