# Implementation Plan — データスコープ権限（data-scope-permission）

逐次実施（1→2→3→4→M）。**前提: P1〜P6 完了後・他 spec と並行させない**
（ほぼ全 ApiController の読み経路に触れるため）。詳細は design.md 参照。

- [x] 0. spec ドキュメント
  - **Objective**: 本ディレクトリ3ファイル。
  - **Demo**: レビュー可能な状態でコミットされている。

- [x] 1. 読み経路の棚卸し
  - **Objective**: design.md 3章の表を実コードと突き合わせて確定させる（エンドポイント×
    スコープ要否×適用方法。適用外の根拠も明記）。
  - **実装ガイダンス**: `@GetMapping` を全 ApiController から列挙し表へ。案件の従属扱いの
    妥当性もここで確定。コード変更なし・表の確定版をコミット。
  - **テスト要件**: なし（調査タスク）。
  - **Demo**: 表が確定し、行数がタスク3/4の適用・テスト件数の根拠になっている。

- [x] 2. スコープ解決基盤
  - **Objective**: config シード・`DataScopeService`（リクエストスコープキャッシュ付き）・
    適用2パターンのヘルパ。
  - **実装ガイダンス**: design.md 1〜2章。IN 空リストの空ページ即返し・404 秘匿の
    `BusinessException.of(404, "error.scope.notFound")`。i18n ×4言語。
  - **テスト要件**: `DataScopeServiceTest`（allowed 集合の解決5ケース＋isScoped 4象限）。
  - **Demo**: curl（営業・設定true）で allowedContractIds に担当契約＋未帰属が入る。

- [x] 3. コア4領域への適用（契約・要員・顧客・提案）
  - **Objective**: 棚卸し表のコア行へ2パターンを機械的に適用する。
  - **実装ガイダンス**: design.md 2〜3章。ページ件数もスコープ後で数える。
  - **テスト要件**: `DataScopeIntegrationTest` — 営業2人データで各領域の
    見える/見えない/他人詳細404/無効時全件（代表2領域は4象限フル）。
  - **Demo**: 営業Aでログイン→契約一覧が担当+未帰属のみ・営業Bの契約詳細URLが404。
    設定 false に戻すと全件。管理者は常に全件。

- [x] 4. 周辺適用（エクスポート・セレクト・見積・営業成績強調）
  - **Objective**: Excel/CSV・セレクト用途 API・見積（P4）への適用と、営業成績の自分行強調。
  - **実装ガイダンス**: design.md 3章。エクスポートは画面と同一スコープ（同じ allowed 集合を
    使う——別実装禁止）。
  - **テスト要件**: エクスポートのスコープ一致2ケース＋セレクト1ケース＋見積1ケース。
  - **Demo**: 営業Aの契約Excel出力に担当外が含まれない。営業成績で自分の行がハイライト。

- [x] M. 統合回帰
  - **Objective**: 回帰確認と文書更新。
  - **テスト要件**: `mvn test` 全緑（既定 false の完全後方互換を含む）。
  - **Demo**: 設定 ON/OFF を切り替えながら営業・管理者・HR で一巡し、
    `.kiro/specs/README.md` の状態列を更新。
