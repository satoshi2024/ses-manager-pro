# 給与情報の外部連携（freee人事労務） 要件定義

## 方針

本システムでは日本の税・社会保険・年末調整を自社実装せず、freee人事労務を計算・法令対応の正本として利用する。SES Manager Pro は要員の対応付けと、権限を持つ利用者向けの明細参照だけを提供する。BP（協力会社）は対象外とし、既存の `t_bp_payment` を継続利用する。

## 要件

1. 管理者は freee の OAuth2 認可を開始、接続状態を確認し、接続解除できる。client secret と token は環境変数/暗号化保管とし、画面・ログに平文を出さない。
2. 管理者または HR は freee 従業員と `t_engineer` を一意に対応付け、解除・再設定できる。`employment_type = BP` は一覧・参照対象から除外し、未設定は要確認と表示する。
3. 管理者または HR は指定年月の給与・賞与明細を freee API から都度取得し、支給合計・控除合計・差引支給額・項目明細を読み取り専用で表示できる。給与金額、口座、扶養情報は本システムに永続化しない。
4. 営業、マネージャー、未認証利用者は本機能へアクセスできない。参照 API は `Cache-Control: no-store` とし、監査ログに操作種別・対象年月・結果だけを記録する。
5. OAuth state、token 有効期限、401/403/429/5xx/タイムアウトを安全に処理し、外部障害時は既存業務を停止させず日本語の操作可能なエラーを返す。

## 受入基準

- テスト用 freee tenant で接続→従業員対応付け→指定月の給与/賞与表示が行える。
- BP が表示・取得・対応付けのすべてから除外される。
- DB ダンプに給与金額・token 平文が含まれない。
- 管理者/HR 以外のページ/API は 403、未認証はログインへ遷移する。
