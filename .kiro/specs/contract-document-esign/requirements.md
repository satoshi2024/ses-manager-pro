# 契約書テンプレート・電子署名 要件

## 目的
契約情報から改ざん検知可能な PDF を生成し、CloudSign で安全に締結できるようにする。

## 要件
1. 管理者は契約形態別の HTML テンプレートを作成・編集・無効化できる。JavaScript、iframe、外部 URL は拒否する。
2. 固定された契約変数を HTML エスケープして置換し、未知の値を空文字にして PDF を生成する。生成時の本文・テンプレート版・SHA-256 を保存する。
3. 契約担当者は宛先を明示確認して下書きを CloudSign に送信できる。送信中、確認中、締結済、却下、取消、エラーを追跡する。
4. 締結済 PDF と合意締結証明書を保存し、権限を検証した利用者だけが no-store でダウンロードできる。
5. 権限はテンプレート CRUD=管理者、生成/送信=管理者・営業・マネージャー、HR=参照のみとする。
